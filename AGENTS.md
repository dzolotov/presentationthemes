# AGENTS

## Общие сведения
- Базовая сетка: каждая страница фиксирована на 960×540px, основная гарнитура — `Roboto`.
- Большинство рабочих шаблонов (`section.page*`, `section.second_page`, `section.third_page`, `section.course_page` и др.) наследуют фон `image11.png` с покрытием `background-size: cover`.
- Общие стили текста, списков, таблиц и кода описаны в `section.page`; остальные шаблоны переопределяют позиционирование и оформление, сохраняя кегли и шрифты.
- Для печати есть корректировки в двух блоках `@media print`; абсолютные элементы (аватары, карточки, стрелки) возвращаются в поток для верной отрисовки PDF.

## Шаблоны страниц
### `section.first_page`
- **Назначение:** Титульный слайд.
- **Особенности:** Крупный `h1`, плашка-`h2`, декоративная полоса (`image5.png`) и логотип в углу (`image4.png`). Все элементы расположены абсолютными координатами.
- **Советы:** Используйте только для открытия презентации; подбирайте короткий подзаголовок, чтобы поместился в кнопку.

### `section.second_page`
- **Назначение:** Основной тезис или девиз прямо после титула.
- **Особенности:** Компактный бейдж (`h1`) с красной точкой, крупный `h3`, две иконки (глаза и динамик) у нижнего края.
- **Советы:** Текст в `h3` лучше ограничивать 2–3 строками, иначе он выходит за отведённую область.

### `section.third_page`
- **Назначение:** Представление спикера.
- **Особенности:** Круглый аватар (277×273px), опциональный фон `additional-image`, карточка с `h1`, `h2` и списком справа.
- **Советы:** Подготовьте изображение аватара в квадрате, чтобы `object-fit: cover` не обрезал важные детали.

### `section.page_rules`
- **Назначение:** Перечень правил/ожиданий.
- **Особенности:** Четыре строки (`h3`–`h6`) шириной 260px, каждая с собственной иконкой из GitHub.
- **Советы:** Держите формулировки короткими; переносы ухудшают читаемость при абсолютном позиционировании.

### `section.course_page`
- **Назначение:** Карта курса или дорожная карта.
- **Особенности:** Сетка 4×N для `<li>` карточек, состояние `.active` подсвечивается фиолетовым, псевдоэлементы `.start`/`.finish` добавляют маркеры.
- **Советы:** Заполняйте элементы короткими названиями модулей; при необходимости переносите длинные цепочки на две строки.

### `section.page_route`
- **Назначение:** Линейный маршрут/дорожка шагов.
- **Особенности:** Вертикальный список карточек с мягкими бордерами; печатный режим превращает сетку в блочный список.
- **Советы:** Используйте по одному предложению на шаг, чтобы не ломать высоту элементов.

### `section.page_twocolumn`
- **Назначение:** Сравнительная таблица в две колонки.
- **Особенности:** Контейнер `.table-container`, шапка `.column-headers` с двумя наборами `h1/h2`, таблица с крупными отступами.
- **Советы:** В Markdown создайте таблицу с нужным числом строк; каждый столбец можно заполнять списками или текстом.

### `section.page`
- **Назначение:** Универсальный текстовый слайд.
- **Особенности:** Отступы `120px/48px`, кнопка внизу слева, унифицированные списки (фиолетовые буллеты и нумерация), стили для `pre/marp-pre` с `Fira Code`, поддержка масштабирования через классы `code-scale-XX`.
- **Советы:** Для CTA-кнопки используйте `<button>`; при многострочном тексте следите за высотой, чтобы не наезжать на нижний отступ.

### `section.page_section`
- **Назначение:** Разделительный слайд.
- **Особенности:** Белые `h1`/`h2` на общем фоне, декоративная линия `image5.png` под заголовком.
- **Советы:** Хорош для коротких названий разделов; избегайте длинных фраз.

### `section.page_question`
- **Назначение:** Вопрос аудитории.
- **Особенности:** Белый `h1`, декоративная линия, иллюстрация `questions.png` ближе к низу.
- **Советы:** Можно добавлять один вопрос или список вопросов через Markdown (`<ul>`/`<ol>`), но учитывайте абсолютное позиционирование картинки.

### `section.page_final`
- **Назначение:** Финальный слайд/контакты.
- **Особенности:** Фон `image36.png`, основной заголовок и три блока (`h4`–`h6`) с иконками (контакты, ресурсы, действия).
- **Советы:** Разместите ключевые контакты или ссылки в этих трёх блоках.

### `section.page_image`
- **Назначение:** Слайд с одной иллюстрацией.
- **Особенности:** Изображение центрируется и ограничено высотой 338px, текст располагается поверх (если нужен).
- **Советы:** Подходит для диаграмм; при необходимости добавляйте подписи через текстовые элементы выше/ниже.

### `section.page_fullscreen_image`
- **Назначение:** Полноэкранная картинка без текста.
- **Особенности:** Изображение растягивается внутри отступа 48px, заголовки скрыты.
- **Советы:** Текст стоит встроить в саму картинку или выбрать другой шаблон.

### `section.page_with_qimage`
- **Назначение:** Текст с небольшим изображением справа (например, вопрос).
- **Особенности:** Текстовые элементы ограничены шириной 728px; картинка высотой до 380px фиксируется справа.
- **Советы:** Используйте лёгкие PNG/SVG, чтобы не перегружать правую колонку.

### `section.page_image_left`
- **Назначение:** Изображение слева, текст справа.
- **Особенности:** Картинка фиксирована 350px ширины и центрируется по высоте; текст смещается на 420px.
- **Советы:** Подходит для диаграмм из `generated/resources`; для других изображений заранее подберите правильный аспект.

### `section.page_image_right`
- **Назначение:** Зеркальный вариант с картинкой справа.
- **Особенности:** Те же размеры и правила, но привязка к правому краю.
- **Советы:** Используйте, если текст логичнее читать слева направо.

### `section.page_image_top`
- **Назначение:** Иллюстрация в верхней части, текст внизу.
- **Особенности:** Изображение занимает ширину слайда, текстовые элементы идут ниже; поддержка диаграмм через специальные селекторы.
- **Советы:** Подходит для больших скриншотов с подписью.

### `section.page_image_bottom`
- **Назначение:** Текст сверху, изображение у нижней границы.
- **Особенности:** Картинка прижата к низу, текст занимает верхнюю область.
- **Советы:** Хорошо работает для выводов с подтверждающей диаграммой снизу.

### `section.page_cards`
- **Назначение:** Две карточки с буллетами.
- **Особенности:** `h2` превращаются в серые плашки, списки позиционируются внутри карточек.
- **Советы:** Каждый список ограничивайте 4–5 пунктами, чтобы текст не выходил за границы блока.

### `section.page_cards_image`
- **Назначение:** Три карточки с изображениями и списками.
- **Особенности:** Реальные `<img>` скрываются, псевдоэлементы `::before/::after` добавляют серый фон и картинку (Unsplash ссылки).
- **Советы:** Для офлайн-сборок замените URL на локальные изображения; оставляйте короткие заголовки.

### `section.page_cards_image_four`
- **Назначение:** Четыре компактные карточки с картинками сверху.
- **Особенности:** Меньший кегль (12px), изображения и фоновые блоки создаются псевдоэлементами.
- **Советы:** Хорошо подходит для презентации ролей команды или преимуществ продукта.

### `section.page_process_steps`
- **Назначение:** Последовательность из четырёх шагов.
- **Особенности:** `h2` превращаются в стрелки с градиентом; под каждым шагом расположен список-описание.
- **Советы:** Ограничьте текст в `h2` 2–3 словами и держите описания лаконичными.

### `section.page_numbered_blocks`
- **Назначение:** Нумерованные карточки с градиентом.
- **Особенности:** Каждый `<p>` получает круглый номер через CSS-счётчик и hover-анимацию.
- **Советы:** Используйте для списков «Top-N», чек-листов или пошаговых инструкций.

### `section.page_two_cases`
- **Назначение:** Сравнение двух кейсов или сценариев.
- **Особенности:** Два градиентных блока (`.block` или `.case-left/right`) с ховер-эффектом, внутри `h2` и `p`.
- **Советы:** Поддерживает также кодовые примеры (`pre`) внутри блоков.

### `section.page_definition`
- **Назначение:** Выделенное определение/цитата.
- **Особенности:** Градиентная «плашка» с курсором-указателем (`::after`), акцент на первом слове (`strong`).
- **Советы:** Подходит для терминов, смысловых акцентов, цитат.

### `section.page_good_bad_example`
- **Назначение:** Пара «хороший / плохой» примеров.
- **Особенности:** Два блока одинаковой структуры с разными градиентами (зелёный и розовый), ховер-анимация.
- **Советы:** Используйте для сравнения best practices vs anti-patterns.

## Дополнительные заметки
- Многие шаблоны используют псевдо-класс `:has()` для перестройки потока (особенно `section.page_image_*` и `section.page_with_qimage`). Убедитесь, что сборщик (Marp CLI, Puppeteer) поддерживает его.
- Карточные шаблоны (`page_cards_image*`) тянут изображения с Unsplash и GitHub. Для офлайн презентаций скачайте их и обновите пути.
- При работе в PDF следите за блоками с абсолютным позиционированием — проверяйте результат после экспорта, чтобы элементы не выходили за границы.
