# Руководство по стилям Marp Theme: otusnew-extended

## Команды компиляции

### Базовая команда
```bash
marp filename.md --theme-set otusnew-extended.css -o output.html --no-stdin && open output.html
```

### Команды для разных типов карточек
- **3-колоночные карточки**: `marp demo-page-cards-image.md --theme-set otusnew-extended.css -o output.html --no-stdin && open output.html`
- **4-колоночные карточки**: `marp demo-page-cards-image-four.md --theme-set otusnew-extended.css -o output.html --no-stdin && open output.html`

---

## Доступные стили страниц

### 1. `page` - Базовая страница
Стандартная страница с текстом и списками.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page -->

# Заголовок страницы

Обычный текст с поддержкой **жирного** и *курсивного* текста.

- Маркированный список
- Второй пункт
- Третий пункт

1. Нумерованный список
2. Второй пункт
3. Третий пункт

`inline код` и блоки кода:

```python
def hello():
    print("Hello, World!")
```
```

### 2. `page_image_left` - Страница с изображением слева
Текст справа, изображение слева.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_image_left -->

# Заголовок с изображением слева

![](path/to/image.png)

Текст располагается в правой части страницы. Поддерживаются все элементы:

- Списки
- **Выделения**
- `Код`

Ширина текста автоматически ограничена для комфортного чтения.
```

### 3. `page_image_right` - Страница с изображением справа
Текст слева, изображение справа.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_image_right -->

# Заголовок с изображением справа

![](path/to/image.png)

Текст располагается в левой части страницы.

- Поддерживаются списки
- Различные **форматирования**
- `Инлайн код`

Изображение автоматически масштабируется.
```

### 4. `page_numbered_blocks` - Нумерованные блоки
Автоматически нумерованные блоки с градиентными фонами и hover-эффектами.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_numbered_blocks -->

# Основные направления развития

Первый блок с автоматической нумерацией

Второй блок с другим градиентом

Третий блок с уникальным цветом

Четвертый блок (поддерживается до 5 блоков)

Пятый блок с автоматическим уменьшением отступов
```

### 5. `page_process_steps` - Процесс со стрелками
Блоки процесса, соединенные стрелками (chevron-дизайн).

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_process_steps -->

# Этапы процесса

Первый этап процесса

Второй этап с переходом

Третий этап завершения

Финальный этап
```

### 6. `page_two_cases` - Сравнение двух кейсов
Два блока для сравнения сценариев (серые карточки).

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_two_cases -->

# Сравнение двух кейсов

## Кейс 1

Описание первого сценария с подробными характеристиками и особенностями реализации.

## Кейс 2

Описание второго сценария для сравнения с первым вариантом решения.
```

### 7. `page_good_bad_example` - Хороший vs плохой пример
Зеленый и розовый блоки для демонстрации контрастов.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_good_bad_example -->

# Хороший пример – плохой пример

<div class="two-blocks">

<div class="block">
<h2>Хорошая практика</h2>
<p>Описание правильного подхода с обоснованием преимуществ и рекомендациями.</p>
</div>

<div class="block">
<h2>Плохая практика</h2>
<p>Описание неправильного подхода с объяснением недостатков и рисков.</p>
</div>

</div>
```

### 8. `page_definition` - Определение с курсором
Фиолетовый блок определения с анимированным курсором.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_definition -->

# Определение

**Термин** – подробное описание понятия с объяснением ключевых характеристик и области применения в контексте предметной области.
```

### 9. `page_with_qimage` - Страница с квадратным изображением
Специальный layout для квадратных изображений.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_with_qimage -->

# Заголовок с квадратным изображением

![](path/to/square-image.png)

Текст для страниц с квадратными изображениями. Автоматическое позиционирование и масштабирование.
```

### 10. `page_twocolumn` - Двухколоночная страница
Контент в двух колонках.

**Пример:**
```markdown
---
marp: true
theme: otusnew-extended
---

<!-- _class: page_twocolumn -->

# Двухколоночная страница

<div class="column1">

## Левая колонка

- Первый пункт
- Второй пункт
- Третий пункт

</div>

<div class="column2">

## Правая колонка

- Другой контент
- Дополнительная информация
- Заключение

</div>
```

## Специальные возможности

### Типография
- **Основной текст**: 18px Roboto
- **Списки**: 16px с фиолетовыми маркерами (#8b5cf6)
- **Инлайн код**: 16px Roboto Mono
- **Блоки кода**: 14px Roboto Mono

### Интерактивность
- Hover-эффекты на блоках
- Анимированный курсор в определениях
- Плавные переходы и тени
- Градиентные фоны

### Адаптивность
- Автоматическое масштабирование изображений
- Оптимальная ширина текста для чтения
- Адаптивные отступы для разного количества блоков

---

## Рекомендации по использованию

1. **Выбор стиля**: Используйте подходящий стиль в зависимости от типа контента
2. **Изображения**: Оптимизируйте размер и формат изображений
3. **Текст**: Соблюдайте оптимальную длину абзацев
4. **Структура**: Используйте заголовки для лучшей навигации
5. **Тестирование**: Проверяйте результат в браузере после компиляции

## Технические требования

- **Marp CLI** последней версии
- **Файл темы**: `otusnew-extended.css`
- **Браузер**: Современный браузер с поддержкой CSS3
- **Изображения**: Форматы PNG, JPG, SVG
